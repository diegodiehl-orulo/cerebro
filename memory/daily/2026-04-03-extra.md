## LIÇÕES APRENDIDAS

### 1. MODELO POR TÓPICO NÃO É NATIVO NO OPENCLAW
- Perguntado se dá para definir LLM por tópico do Telegram — não é nativo.
- A forma determinística é: **bot separado por LLM**, não tópico.
- **Regra:** Quando Diego pedir "LLM diferente por contexto", recomendar bot separado. Tópico é separação de assunto, não de modelo.

### 2. BUG DE ROUTING: LARISSA RESPONDENDO COMO MORFEU
- Sessão `larissa:main` estava respondendo com Claude Sonnet (modelo do Morfeu) em vez de M2.7.
- Causa: modelo configurado incorretamente no agent larissa.
- Resolução: config-patch corrigindo primary model.
- **Regra:** Ao iniciar sessão em agent não-morfeu, verificar qual modelo está ativo. Não assumir que binding = modelo correto.

### 3. groupAllowFrom ≠ channels.telegram.groups
- Tentei autorizar o grupo via `groupAllowFrom` — config errado, bots ignoraram as mensagens.
- O campo correto no OpenClaw é `channels.telegram.groups` com o chat_id do grupo.
- **Regra:** Para autorizar grupos Telegram, sempre usar `channels.telegram.groups`, nunca `groupAllowFrom` (que serve para usuários, não grupos).

### 4. GRUPO TELEGRAM NÃO PODE SER CRIADO PELO BOT
- Tentar criar grupos ou topics via API sem ter o chat_id causa "chat not found".
- Fluxo correto: Diego cria o grupo + ativa tópicos + adiciona bots como admin → me passa o ID → eu configuro.
- **Regra:** Setup de grupo Telegram é sempre humano primeiro. Só então configurar roteamento.

### 5. requireMention: true É NECESSÁRIO EM GRUPOS
- Sem `requireMention: true`, bots respondem a toda mensagem no grupo = spam.
- Com a flag ativa, só respondem quando mencionados diretamente.
- **Regra:** Todo grupo Telegram deve ter `requireMention: true` por padrão. Só desativar se Diego pedir explicitamente.
