# Memória — 07/04/2026

## tl;dv Integration — COMPLETA ✅

### O que foi construído hoje

**Scripts principais:**
- `integrations/tldv/client.py` — Cliente Python da API tl;dv v1alpha1. Autenticação via `TLDV_API_KEY` env. Endpoints: `health`, `list_meetings` (page/pageSize), `get_meeting`, `get_transcript`, `get_notes`.
- `integrations/tldv/collector.py` — Lista reuniões via API, salva em `memory/meetings/raw/{id}.json`. Ledger idempotente. Lock file.
- `integrations/tldv/enricher.py` — Para cada raw: baixa transcript + notes, salva em `transcripts/` + `notes/`, gera `normalized/{id}.json`. Transcript 204 → `pending`, vazio → `unavailable`.
- `integrations/tldv/analyzer.py` — LLM analysis via `openclaw agent --agent morfeu`. Gera `analysis/{id}.json` + `.md`. 11 blocos: resumo, contexto, decisões, pendências, tarefas, perguntas críticas, riscos, oportunidades, sugestões doc, memórias, próximo passo.
- `integrations/tldv/persister.py` — Extrai decisões/lições/riscos/context de análises → arquivos de memória + patches em `docs/meeting-derived-updates/`. SHA256 dedup.
- `integrations/tldv/task_generator.py` — Gera tarefas estruturadas a partir de análises. 4 tipos: EXPLICITA / FOLLOW_UP / DECISION_PENDING / INFO_NEEDED. Saídas: `tasks/generated-from-meetings/{id}.json+md`, `tasks/consolidated/all_tasks.json`, `tasks/consolidated/by_owner.json`.
- `integrations/tldv/pipeline.py` — Orquestrador. Estágios: collect / enrich / analyze / persist / digest.
- `integrations/tldv/queue_processor.py` — Consome fila webhook, processa por reunião. `--meeting-id ID` para reprocessar.
- `integrations/tldv/webhook_server.js` — Receptor webhooks na porta 18788. Eventos: MeetingReady / TranscriptReady. SHA256 dedup por eventType+meetingId+timestamp.
- `integrations/tldv/webhook_server.sh` — Inicia/monitora webhook server como service.
- `scripts/tldv_daily.sh` — Rotina diária (digest + task generation). Cron: 08h daily.
- `scripts/tasks_to_excel.py` — Gera Excel com abas: Resumo / Tarefas / Críticas / Sem Dono / Sem Prazo / Por Responsável. Usa openpyxl.

### Bugs encontrados e corrigidos
1. **API pagination:** `/meetings` usa `page`+`pageSize` — NÃO `offset`+`limit`
2. **Transcript 204:** API retorna 204 quando transcript ainda processando — client retorna `None`, enricher marca `pending`
3. **KeyError confidence:** analyzer.py linha 215 — `decisions[i]["confidence"]` quebrava se LLM não retornava campo. Corrigido para `.get("confidence", "uncertain")`
4. **find -exec rm:** Não usar `find DIR -name "X" -exec rm {}` — recursa em subdirs. Usar `find DIR -maxdepth 1 -name "X" -delete`
5. **GENERATED_DIR = ANALYSIS_DIR:** task_generator tinha `GENERATED_DIR = ANALYSIS_DIR` — sobrescrevia analysis JSONs. Corrigido para `WORKSPACE / "tasks/generated-from-meetings"`
6. **Batch runner acumular:** processos duplicados. Matados todos,只剩1 runner

### Limitações técnicas
- Gateway OpenClaw serializa todas as requisições em 1 lane — `openclaw agent --agent morfeu` compete com Morfeu. Paralelização real precisa de múltiplas lanes ou agentes isolados.
- Análise por reunião: ~90-300s (gateway serial + LLM). 148 reuniões = ~19h sequencial.
- Transcript 204: reuniões pendentes precisam de re-enrich later.
